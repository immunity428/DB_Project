08

編集・削除の仕組み

編集：POST /edit/:id
1books を UPDATE（タイトル・年・ページ・購入者）
2book_genres を全削除 → 入力で作り直し

外部キー制約を守るため 子テーブルから順に削除。

① history を削除
↓
② book_genres を削除
↓
③ books 本体を削除
Key SQL Pattern
09

「最新の状態」を求めるサブクエリ

1冊の書籍に複数の履歴（貸出・返却・破棄…）がある。一覧・詳細で使う“最新1件”を相関サブクエリで抽出。

LEFT JOIN (
    SELECT h.book_id, h.action
    FROM history h
    WHERE h.history_id = (
        SELECT h2.history_id
        FROM history h2
        WHERE h2.book_id = h.book_id
        ORDER BY h2.action_year DESC,
                 h2.history_id  DESC
        LIMIT 1          -- 最新1件に絞る
    )
) latest_history
  ON b.book_id = latest_history.book_id
並び替えaction_year（年）→ history_id の降順で最新を決定
1件に限定LIMIT 1 で各書籍の最新履歴だけを取得
共通部品一覧・詳細・購入者・ジャンル 各所で再利用
